sign_language.models.loader module

Model loading and initialisation.

Loads EfficientNet and Landmark MLP checkpoints from disk, selects the compute device, and initialises the MediaPipe hand landmarker.

The EfficientNet checkpoint source is selected by settings.deploy_target:

  • "local": read from settings.model_path on disk.

  • "onprem": fetch the version aliased @champion from the self-hosted MLflow model registry at settings.mlflow_tracking_uri.

  • "azure": fetch the latest promoted version from the Azure ML model registry under settings.azure_subscription_id / settings.azure_resource_group / settings.azure_workspace.

Both remote paths cache downloaded checkpoints under settings.model_cache_dir keyed by version, so repeated container restarts do not re-download unchanged models.

sign_language.models.loader.download_latest_model_azure(model_name: str, download_dir: Path) → Path[source]

Download the latest registered EfficientNet checkpoint from Azure ML.

Prefers the version explicitly tagged promoted=true, selecting the one with the highest f1_macro (with accuracy as tiebreaker). Falls back to label="latest" when no promoted version exists.

Downloaded assets are cached under download_dir keyed by version number. On subsequent startups the cached file is reused if the version has not changed.

Parameters:

  • model_name – Registry name, e.g. "ngt-sign-language".

  • download_dir – Local directory to cache the downloaded checkpoint.

Returns:

Path to the downloaded .pth file.

Raises:

  • RuntimeError – If Azure ML dependencies are not installed, or if any required Azure settings (subscription ID, resource group, workspace) are missing.

  • FileNotFoundError – If no .pth file is found in the download directory after the download completes.

sign_language.models.loader.download_latest_model_mlflow(model_name: str, download_dir: Path, alias: str = 'champion') → Path[source]

Download the EfficientNet checkpoint aliased @<alias> from MLflow.

Resolves models:/<model_name>@<alias> against the MLflow registry configured by settings.mlflow_tracking_uri. Downloads the artifact directory to a version-keyed local cache; on subsequent startups the cached .pth file is reused if the aliased version has not changed.

The MLflow server is expected to run with --serve-artifacts (the BUaS on-prem reference configuration), so the client only needs the tracking URI — no MinIO / S3 credentials.

Parameters:

  • model_name – Registered model name, e.g. "ngt-sign-language".

  • download_dir – Local directory to cache the downloaded checkpoint.

  • alias – Registry alias to resolve. Defaults to "champion".

Returns:

Path to the downloaded .pth file.

Raises:

  • RuntimeError – If MLflow is not installed, or if MLFLOW_TRACKING_URI is not configured.

  • FileNotFoundError – If no .pth file is present in the downloaded artifact directory.

sign_language.models.loader.resolve_efficientnet_path() → Path[source]

Return the EfficientNet checkpoint path to load at startup.

Dispatches on settings.deploy_target:

  • "local": returns settings.model_path unchanged.

  • "onprem": downloads the version aliased @champion from the self-hosted MLflow registry via download_latest_model_mlflow().

  • "azure": downloads the latest promoted version from the Azure ML registry via download_latest_model_azure().

Returns:

Path to the EfficientNet .pth checkpoint.

Raises:

  • RuntimeError – If the selected target’s required dependencies or settings are missing.

  • FileNotFoundError – If a remote download completes but no .pth file is found in the cache directory.

  • ValueError – If settings.deploy_target is set to an unknown value (should be unreachable due to the Literal constraint).

sign_language.models.loader.resolve_landmark_path() → Path | None[source]

Return the Landmark MLP checkpoint path to load at startup, if available.

Dispatches on settings.deploy_target, mirroring resolve_efficientnet_path():

  • "local": returns settings.lm_model_path unchanged. If the file doesn’t exist, load_landmark_mlp will skip it gracefully — same behaviour as before.

  • "onprem" / "azure": attempts to download the version aliased @champion from the corresponding registry. Returns None if the model isn’t registered, the registry is unreachable, or the download fails — the landmark MLP is a fallback, not a hard requirement, so we degrade gracefully rather than blocking app startup.

Returns:

Path to the Landmark MLP .pth file, or None if no checkpoint could be resolved.

sign_language.models.loader.load_all(model_path: Path | None = None, lm_model_path: Path | None = None, landmarker_path: Path | None = None) → LoadedModels[source]

Load every model and return them in a single LoadedModels container.

When model_path is None, resolve_efficientnet_path() is called, which dispatches on settings.deploy_target to either return the local on-disk path, download from MLflow (onprem), or download from the Azure ML registry (azure).

Parameters:

  • model_path – Path to the EfficientNet .pth checkpoint, or None to resolve automatically via resolve_efficientnet_path().

  • lm_model_path – Path to the Landmark MLP .pth checkpoint, or None to resolve automatically via resolve_landmark_path().

  • landmarker_path – Path to the MediaPipe hand_landmarker.task file. Defaults to settings.hand_landmarker_path.

Returns:

A LoadedModels dataclass with all models loaded and ready for inference.
